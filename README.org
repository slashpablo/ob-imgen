#+title: ob-imgen — Org-babel image generation
#+author: Pablo Munoz Haro

Generate images from text prompts inside Org mode source blocks.

#+begin_src org
,#+begin_src imgen :model gpt-image-1 :quality medium :file fox.png
A red fox sitting on a snowy mountain at sunset
,#+end_src
#+end_src

* Features

- Works with OpenAI models: =dall-e-2=, =dall-e-3=, =gpt-image-1=
- Images display inline automatically after generation
- Configurable output directory, quality, size, and style
- API key via environment variable or custom function (e.g. auth-source)

* Installation

** MELPA (recommended)

#+begin_src emacs-lisp
(use-package ob-imgen
  :after org)
#+end_src

** Manual

Clone the repo and add it to your load path:

#+begin_src shell
git clone https://github.com/slashpablo/ob-imgen.git ~/path/to/ob-imgen
#+end_src

#+begin_src emacs-lisp
(add-to-list 'load-path "~/path/to/ob-imgen")
(require 'ob-imgen)
#+end_src

* Configuration

Set your OpenAI API key:

#+begin_src shell
export OPENAI_API_KEY="sk-..."
#+end_src

Or customize the key retrieval function:

#+begin_src emacs-lisp
(setq ob-imgen-api-key-fn
      (lambda ()
        (auth-source-pick-first-password :host "api.openai.com")))
#+end_src

Optional settings:

#+begin_src emacs-lisp
(setq ob-imgen-default-model "gpt-image-1")     ; default: "dall-e-3"
(setq ob-imgen-default-size "1024x1024")         ; default: "1024x1024"
(setq ob-imgen-output-directory "~/org/images/") ; default: nil (same dir as org file)
(setq ob-imgen-display-inline t)                 ; default: t
#+end_src

* Header arguments

| Arg          | Description                                          | Default       |
|--------------+------------------------------------------------------+---------------|
| =:model=     | Model name                                           | =dall-e-3=    |
| =:size=      | Image dimensions                                     | =1024x1024=   |
| =:quality=   | Quality (dall-e-3: standard/hd; gpt-image-1: low/medium/high) | model-dependent |
| =:style=     | vivid or natural (dall-e-3 only)                     | =vivid=       |
| =:file=      | Output filename                                      | auto-generated |
| =:output-dir=  | Directory for auto-generated files                  | org file dir  |
| =:style-guide= | Style guide preamble name (see Style guides section) | none          |

* Style guides

You can define reusable style preambles that get prepended to your prompts
automatically.

1. Set the style guide directory:

#+begin_src emacs-lisp
(setq ob-imgen-style-guide-directory "~/org/image-styles/")
#+end_src

2. Create a style guide file, e.g. =~/org/image-styles/editorial.md=:

#+begin_example
Contemplative editorial illustration, literary rather than fantasy-heavy,
warm light, reverent and intelligent, muted earth tones
#+end_example

3. Reference it with =:style-guide=:

#+begin_src org
,#+begin_src imgen :style-guide editorial :model gpt-image-1
A human figure formed from earth in a garden with flowing water
,#+end_src
#+end_src

The style guide text is prepended to the block body, so the actual prompt
sent to the API becomes:

#+begin_example
Contemplative editorial illustration, literary rather than fantasy-heavy,
warm light, reverent and intelligent, muted earth tones

A human figure formed from earth in a garden with flowing water
#+end_example

Both =.md= and =.txt= extensions are supported.  If no =:style-guide= header
is given, behavior is unchanged.

* Examples

Basic usage (default model):

#+begin_src org
,#+begin_src imgen
A watercolor painting of a quiet library
,#+end_src
#+end_src

With gpt-image-1 and explicit output:

#+begin_src org
,#+begin_src imgen :model gpt-image-1 :quality high :size 1024x1536 :file hero.png
Editorial illustration of a sunrise over ancient ruins
,#+end_src
#+end_src
